GitHub Pages 網頁安全架構建置指南

0. 第零階段：帳號對接與加密地基

🔐 這是所有程序的邏輯起點：

1. 帳號對接：定義 GitHub User ID 與對應 Repo。

2. SHA-256 雜湊 (Hash)：將密碼轉為不可逆雜湊值存入 pass.json。禁止存儲明文，這是安全底線。

1. 雙倉庫 (Repo) 分離戰略

透過物理隔離達成「外殼公開、數據私有」：

  • Public Repo：負責 index.html 與 UI 背景。
  • Private Repo：負責數據與密碼雜湊。

2. 關鍵：雙倉庫目錄結構詳解

GitHub // 您的帳號 ┣━ 🌐 Public Repo (您的.github.io) ┃ ┣ 📂 background/ // UI 背景圖 ┃ ┗ 📄 index.html // 金庫入口 ┃ ┗━ 🔒 Private Repo (私有數據庫) ┣ 📄 pass.json // 密碼雜湊 ┗ 📄 source.html // 私密書籤

3. 核心：GitHub API Token 轉碼程序

GitHub 會掃描並廢除明文 Token (ghp_)。必須先透過 btoa() 程序進行編碼。這段註解建議直接編入 index.html 方便日後維護：

<!--〔 Edit data 〕Bookmark 2026/03/08 @ GitHub --> <!--〔 GitHub API key 〕ghp_xxxx (開發暫存) --> <!--〔 程序名稱 〕btoa("ghp_xxxx") : Base64 編碼程序 --> <!--〔 GitHub API key base64 code 〕WjJocF... --> // 轉碼後將結果貼入下方碎裂化變數中，並刪除含 ghp_ 的註解

4. Token 碎裂化打散技術

將 btoa() 產出的編碼字串物理拆分。GitHub 掃描器無法拼湊出具備特徵的字串，從而達成「源碼公開，權杖安全」。

const k_p1 = "WjJocF"; // 碎片 A const k_p2 = "0ZW50"; // 碎片 B const k_p3 = "...=="; // 碎片 C

5. 全域透明 UI 邏輯

將 index.html 的容器設為 transparent !important。這樣不論內容多長，背景都能穩固穿透顯示，解決下拉漏白問題。

6. Session 持久化恢復 (Reload)

整合 sessionStorage。讓使用者在重新整理頁面時無需重打密碼，提升流暢度。
