GitHub Pages 網頁安全架構建置指南

本文件旨在提供一個標準化的 GitHub Pages 私密化建置方案，確保 API 密鑰不外洩，並維持視覺與操作的穩定性。

0. 第零階段：帳號對接與加密地基

🔐 這是所有程序的邏輯起點：

1. 帳號對接：明確定義你的 GitHub User ID 與 Repo 名稱，作為 API 調用的唯一標記。

2. SHA-256 雜湊 (Hash)：

  • 將密碼轉換為不可逆的 SHA-256 雜湊值。
  • 禁止存儲明文密碼：這是保護帳號的第一道防線。

1. 雙倉庫 (Repo) 分離戰略

透過物理隔離來達成「看的到門，看不到內容」的效果：

  • Public Repo (入口網頁)：負責渲染 index.html 與靜態 UI。
  • Private Repo (核心數據)：存放 source.html 與 pass.json。

2. 關鍵：雙倉庫目錄結構詳解

採用「懶人路徑同步法」，確保兩邊資源路徑完全一致，避免 404 報錯：

GitHub // 您的帳號 ┣━ 🌐 Public Repo (您的.github.io) ┃ ┣ 📂 background/ // 🖼️ UI 背景圖 (直接渲染) ┃ ┣ 📂 ext_file/ // ⚙️ 外部插件與 JS ┃ ┣ 📂 favicon/ // ☁️ 清晰版圖標 ┃ ┣ 📂 index_files/ // 🎨 選單 CSS 與樣式 ┃ ┗ 📄 index.html // 🔑 金庫大門 (解鎖邏輯) ┃ ┗━ 🔒 Private Repo (私有數據庫) ┣ 📂 background/ // 📦 懶人同步備份 (路徑一致) ┣ 📂 ext_file/ // 📦 懶人同步備份 ┣ 📄 pass.json // 🔐 密碼 SHA-256 雜湊 ┗ 📄 source.html // 🔖 私密書籤 (解鎖後抓取)

3. API Key 安全轉碼 (Base64)

為了閃避 GitHub 的掃描機器人，必須將 PAT (Token) 轉換為 Base64 字串，隱藏 ghp_ 前綴特徵。

4. Token 碎裂化打散技術

將編碼後的字串進行物理拆分（例如拆成三段變數），確保即便掃描器讀取到代碼，也無法還原成有效的 Token。

const k_p1 = "Z2hwX3"; // 碎片 A const k_p2 = "h4eHgx"; // 碎片 B const k_p3 = "NTV6dw=="; // 碎片 C

5. 全域透明 UI 邏輯

為了解決「背景不動、內容捲動」時出現的漏白問題，必須將 index.html 的外層容器設為 transparent !important。

6. Session 持久化恢復 (Reload)

利用 sessionStorage 機制。當使用者重新整理頁面時，系統會自動檢查分頁暫存，若有解鎖紀錄則直接顯示，無需重打密碼。
