GitHub Pages 安全金庫無痛建置指南

本指南將引導你完成一個「源碼公開、數據私密」的網頁架構。只要照著程序走，你就能在不碰壁的情況下，成功建立屬於自己的安全空間。

0. 第零階段：帳號對接與加密地基

⚠️ 這是最關鍵的地基作業：

1. 帳號精確對接：請先確認你的 GitHub ID 與 Private Repo 名稱。API 調用時路徑若錯一字，後續將全盤失效。

2. SHA-256 密碼雜湊 (Hash)：

  • 禁止存儲明文：將你的登入密碼轉為 SHA-256 Hash。
  • 驗證邏輯：你的網頁只負責比對「雜湊後的字串」，這樣即便 pass.json 公開，也沒人知道你的原始密碼。

1. 雙倉庫 (Repo) 分離戰略

採用「分流管理」：入口在明處，數據在暗處。

  • Public Repo (入口)：放置 index.html (你的門面與解鎖程式)。
  • Private Repo (數據)：放置 source.html (真正的私密內容) 與 pass.json (密碼比對檔)。

2. 關鍵：雙倉庫目錄結構詳解

維持以下目錄結構，並確保兩邊「資料夾名稱」一致，達成路徑無痛對接：

GitHub // 您的帳號 ┣━ 🌐 Public Repo (pipi-family.github.io) ┃ ┣ 📂 background/ // UI 背景圖 (index.html 直接讀取) ┃ ┣ 📂 favicon/ // 清晰標籤圖示 ┃ ┗ 📄 index.html // 🔒 入口核心程式 ┃ ┗━ 🔒 Private Repo (my-private-data) ┣ 📂 background/ // 📦 懶人同步 (確保 source.html 預覽不破圖) ┣ 📄 pass.json // 🔐 密碼 SHA-256 雜湊值 ┗ 📄 source.html // 🔖 私密數據 (書籤、筆記等)

3. 核心程序：btoa 轉碼與註解規範

為了閃避 GitHub 的掃描機器人，Token 寫入程式碼前必須先經過「btoa 轉碼」。建議在 index.html 頂端保留開發註解：

<!--〔 Edit data 〕Bookmark 2026/03/08 @ GitHub --> <!--〔 GitHub API key 〕ghp_xxxx (明文暫存，轉碼後務必移除) --> <!--〔 程序名稱 〕btoa("ghp_xxxx") : Base64 編碼程序 --> <!--〔 GitHub API key base64 code 〕WjJocF... (轉碼後的成果) -->

4. Token 碎裂化技術

將 btoa() 得到的編碼結果拆成三段變數。這樣做能讓 GitHub 的自動掃描機制失效，保護你的 API Token 永不被停用。

const k_p1 = "WjJocF"; // 第一段 const k_p2 = "0ZW50"; // 第二段 const k_p3 = "...=="; // 第三段

5. 全域透明 UI 邏輯

為了解決內容注入後的背景漏白問題，請在 index.html 加入以下 CSS 修補：

html, body, #content-main { background-color: transparent !important; }

這能確保不論內容多長，背景圖都能維持 fixed 效果，達到最完美的視覺呈現。

6. Session 持久化恢復機制

透過 sessionStorage 暫存解鎖內容。只要分頁不關閉，使用者重新整理 (F5) 就不需要重複輸入密碼，體驗與一般網頁無異。
