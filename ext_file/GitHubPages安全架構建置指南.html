GitHub Pages 安全金庫建置指南

這是一份標準化建置 SOP，旨在協助您在公開的 GitHub 環境中，透過技術手段達成「源碼公開、數據私密」的目標。

0. 第零階段：帳號對接與加密地基

🔐 這是所有建置的邏輯起點：

1. 帳號對接：定義 GitHub User ID 與目的 Repo，這是程式碼抓取資料的絕對座標。

2. SHA-256 密碼雜湊：

  • 將密碼轉成 SHA-256 雜湊值存入 pass.json。
  • 核心邏輯：網頁只比對「雜湊結果」，不存儲「真實密碼」。

1. 雙倉庫 (Repo) 分離戰略

透過物理隔離達成安全性：入口在明 (Public)，資料在暗 (Private)。

2. 關鍵：雙倉庫目錄結構詳解

維持以下目錄結構，確保兩邊路徑無痛對接，不產生 404 錯誤：

GitHub // 您的帳號 ┣━ 🌐 Public Repo (pipi-family.github.io) ┃ ┣ 📂 background/ // 🖼️ 背景圖 (瀏覽器直接讀取) ┃ ┣ 📂 favicon/ // ☁️ 清晰版分頁圖示 ┃ ┗ 📄 index.html // 🔒 金庫大門 (解鎖邏輯) ┃ ┗━ 🔒 Private Repo (my-private-data) ┣ 📂 background/ // 📦 懶人同步 (預覽 source.html 用) ┣ 📄 pass.json // 🔐 密碼 SHA-256 雜湊 ┗ 📄 source.html // 🔖 私密書籤數據

3. 如何精確取得 GitHub API key (PAT)

這是網頁調用私有資料的「萬能鑰匙」，請務必妥善保管：

取得步驟：
1. 進入 GitHub -> 點擊頭像 -> Settings。
2. 拉到最下方 -> Developer settings。
3. 選擇 Personal access tokens -> Tokens (classic)。
4. 點擊 Generate new token (classic)。
5. 權杖權限勾選 repo 即可。
6. 生成後立即複製，格式應為 ghp_...。

4. 核心：btoa 轉碼程序與碎裂化

GitHub 會自動廢除明文 ghp_。必須透過 btoa() 轉碼並打散，這是保護權杖的關鍵：

如何執行 btoa 程序：
1. 在瀏覽器網頁按 F12 打開「開發者工具」。
2. 點擊 Console (主控台) 標籤。
3. 輸入：btoa("你的ghp_key") 並按 Enter。
4. 複製產出的 Base64 字串（例如：WjJocF...）。

將轉碼結果依據下列註解樣板編入 index.html：

<!--〔 Edit data 〕Bookmark 2026/03/08 @ GitHub --> <!--〔 GitHub API key 〕ghp_xxxx (轉碼完成後請務必刪除此行) --> <!--〔 程序名稱 〕btoa("ghp_xxxx") : Base64 編碼程序 --> <!--〔 GitHub API key base64 code 〕WjJocF... --> // 碎裂化存放範例 const k_p1 = "WjJocF"; const k_p2 = "0ZW50"; const k_p3 = "...==";

5. 全域透明 UI 修補

解決內容注入後的下拉漏白問題：

html, body, #content-main { background-color: transparent !important; }

6. Session 恢復機制

利用 sessionStorage。讓使用者在重新整理 (F5) 時能瞬間恢復解鎖狀態，無需重打密碼。
